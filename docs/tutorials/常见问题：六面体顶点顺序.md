# 常见问题：六面体顶点顺序

在使用 PyVista 进行 3D 可视化时，一个常见的问题是网格显示不正确，可能显示为扭曲的形状而非规则的立方体。这通常是由于六面体单元的顶点顺序不符合 PyVista/VTK 规范导致的。

## 问题描述

在构建结构化网格时，每个六面体单元由8个顶点组成。如果这些顶点的顺序不正确，PyVista 将无法正确地渲染网格，导致可视化结果出现扭曲。

## PyVista/VTK 六面体顶点顺序规范

在 PyVista 和 VTK 中，六面体的顶点顺序有严格规定：

```
    顶面 (逆时针)
    4 -------- 5
    |          |
    |          |
    |          |
    7 -------- 6

    底面 (逆时针)
    0 -------- 1
    |          |
    |          |
    |          |
    3 -------- 2
```

具体顺序要求：

1. 底面四个顶点按逆时针方向排列：0, 1, 2, 3
2. 顶面四个顶点按逆时针方向排列：4, 5, 6, 7
3. 对应的顶点垂直对齐（0与4，1与5，2与6，3与7）

## 正确的实现方式

在构建网格时，确保按以下顺序设置顶点索引：

```python
# 正确的顶点索引顺序
i0 = i * (ny + 1) * (nx + 1) + j * (nx + 1) + k          # 0
i1 = i * (ny + 1) * (nx + 1) + j * (nx + 1) + k + 1      # 1
i2 = i * (ny + 1) * (nx + 1) + (j + 1) * (nx + 1) + k + 1  # 2
i3 = i * (ny + 1) * (nx + 1) + (j + 1) * (nx + 1) + k      # 3
i4 = (i + 1) * (ny + 1) * (nx + 1) + j * (nx + 1) + k      # 4
i5 = (i + 1) * (ny + 1) * (nx + 1) + j * (nx + 1) + k + 1  # 5
i6 = (i + 1) * (ny + 1) * (nx + 1) + (j + 1) * (nx + 1) + k + 1  # 6
i7 = (i + 1) * (ny + 1) * (nx + 1) + (j + 1) * (nx + 1) + k      # 7

vertices = [i0, i1, i2, i3, i4, i5, i6, i7]
```

## 避免问题的建议

1. 在实现网格生成时，仔细参考 PyVista/VTK 的官方文档
2. 编写可视化测试代码，验证网格是否正确显示
3. 注意区分不同可视化库的顶点顺序要求，它们可能不一致
